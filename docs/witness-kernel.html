Overview

Three design constraints

Every decision serves one goal: produce useful security events without creating a surveillance substrate.

Event-based output

Sealed log of discrete events. No continuous video export API.

No biometric code

Face, plate, gait recognition absent from codebase. Not disabled ‚Äî missing.

Quorum export

Raw footage requires N-of-M trustee approval. No single-actor extraction.

How It Works

Enforced constraints

I

Private frame data

RawFrame.data has no public accessor. Modules receive InferenceView only.

II

Token-gated export

export_for_vault() requires BreakGlassToken from quorum authorization.

III

Coarsened timestamps

10-minute buckets at capture. Precise timestamps not stored.

IV

Hash-chained log

Each event includes hash of previous. Tampering breaks verification.

V

Single-use tokens

Break-glass tokens are envelope-bound, consumed on use, expire within bucket.

VI

Forward-only rulesets

ReprocessGuard blocks new rules on historical data.
